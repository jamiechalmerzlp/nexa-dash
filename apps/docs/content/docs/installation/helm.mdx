## Installation using Helm

Helm is a package manager for Kubernetes that makes it easier to deploy and
manage applications on a cluster.

This repository does not currently publish a dedicated NexaDash Helm chart, but
you can still deploy NexaDash with your preferred chart or wrapper by pointing
it at the published container image and using `nexadash` as the release and
namespace name.

### Prerequisites

- [Helm](https://helm.sh/docs/intro/install/)

### Example values

Use the following values as a starting point for your chart:

```yaml title="values.yaml"
image:
  repository: jamiechalmerzlp/nexadash
  tag: latest

service:
  port: 3001

ingress:
  enabled: true
  className: 'traefik'
  annotations:
    # Add any additional annotations as needed
  hosts:
    - host: nexadash.homelab.dev
      paths:
        - path: /
          pathType: ImplementationSpecific
  tls:
    - hosts:
        - 'nexadash.homelab.dev'
        - 'www.nexadash.homelab.dev'
      secretName: homelab-dev-tls
```

If your chart supports directly passing image settings and environment
variables, set the image repository to `jamiechalmerzlp/nexadash` and use the
same `NEXADASH_*` environment variables documented throughout this site.

## Uninstalling the Chart

To uninstall the `NexaDash` deployment:

```bash
helm uninstall nexadash
```

This removes the Helm release. Persistent volumes and other cluster resources
may need to be cleaned up separately depending on the chart you use.
