```yaml
services:
  nexadash:
    image: jamiechalmerzlp/nexadash:latest
    restart: unless-stopped
    privileged: true
    ports:
      - '80:3001'
    environment:
      NEXADASH_SHOW_HOST: 'true'
      NEXADASH_ENABLE_CPU_TEMPS: 'true'
    volumes:
      - type: bind
        source: /
        target: /mnt/host
        read_only: true
```

This long-form bind mount is the recommended default. It ensures NexaDash can
read host OS, storage, and network data from `/mnt/host`, and it avoids Docker
or Portainer silently converting the mount into a named volume.

## Configuration

Config options can optionally set, by using the [environment](https://docs.docker.com/compose/compose-file/#environment) field.

```yaml
services:
  nexadash:
    environment:
      NEXADASH_ENABLE_CPU_TEMPS: 'true'
    # ...
```

## GPU Support

GPU support is available with another image tag and a slightly different config.

```yaml
services:
  nexadash:
    image: jamiechalmerzlp/nexadash:nvidia
    restart: unless-stopped
    privileged: true
    deploy:
      resources:
        reservations:
          devices:
            - capabilities:
                - gpu
    ports:
      - '80:3001'
    volumes:
      - type: bind
        source: /
        target: /mnt/host
        read_only: true
    environment:
      NEXADASH_WIDGET_LIST: 'os,cpu,storage,ram,network,gpu'
```

Use the same long-form bind mount for GPU deployments too.
